## Обзор проекта

Это исследовательский проект (дипломная работа) по предобработке осциллограмм электросети в формате COMTRADE (IEEE Std C37.111) для задачи локализации короткого замыкания (КЗ) с помощью машинного обучения.

Основной файл проекта — Jupyter Notebook `fault_location_preprocessing_pipeline.ipynb`, который содержит полный end-to-end пайплайн обработки одного COMTRADE-файла от сырых данных до готового тензора для нейросети.

### Что делает пайплайн

1. **Парсинг COMTRADE** — читает `.cfg` (метаданные) и `.dat` (осциллограмма).
2. **Фильтрация** — удаление апериодической (DC) составляющей методом центрированного скользящего среднего за период сетевой частоты (20 мс). Альтернативно доступен ФВЧ Баттерворта 4-го порядка, `fc = 25` Гц.
3. **Детекция момента КЗ** — комбинированный критерий скользящих окон RMS (T/4): рост тока `I_post / I_pre > 1.5` и падение напряжения `U_post / U_pre < 0.85`.
4. **Оконирование** — вырезает окно `[-50 мс, +150 мс]` относительно момента КЗ с zero-padding при выходе за границы.
5. **Симметричные составляющие (Фортескю)** — вычисляет мгновенные `I₀, I₁, I₂` и `U₀, U₁, U₂`.
6. **Классификация вида КЗ** — по отношениям `I₂/I₁` и `I₀/I₁` определяет тип: `3ф`, `2ф`, `2ф-З`, `1ф-З`.
7. **Нормализация в p.u.** — приводит токи и напряжения к относительным единицам на основе `LINE_PARAMS` и базовой мощности `S_BASE_MVA`.
8. **Формирование тензора** — собирает `X ∈ R^(12 × L_window)` и целевую переменную `y` (расстояние до КЗ в p.u.).
   - Каналы: `IA, IB, IC` (фазные токи), `I₁, I₂, I₀` (симметричные токи), `UA, UB, UC` (фазные напряжения), `U₁, U₂, U₀` (симметричные напряжения).
   - Вид КЗ **не включается** в тензор. Классификация выполняется отдельным модулем (`classify_fault_type`), и по её результату выбирается одна из 11 моделей ансамбля для оценки расстояния.

## Организация кода

Код организован по ячейкам в логическом порядке:

| Раздел | Содержимое |
|--------|-----------|
| Импорты | `numpy`, `pandas`, `matplotlib`, `scipy`, `struct`, `re`, `dataclasses` |
| Раздел 1 | `find_comtrade_files()`, `inspect_channels()`, `parse_comtrade()`, `plot_phases()` |
| Раздел 2 | `butterworth_highpass()`, `remove_dc_period()`, сохранение отфильтрованного COMTRADE |
| Раздел 3 | `detect_fault_inception()` — комбинированный RMS-критерий по току и напряжению |
| Раздел 4 | Оконирование сигнала |
| Раздел 5 | `fortescue_instantaneous()` — мгновенные симметричные составляющие |
| Раздел 6 | `classify_fault_type()` — классификация вида КЗ (используется для выбора модели ансамбля) |
| Раздел 7 | Нормализация в p.u. по параметрам линии (`LINE_PARAMS`) |
| Раздел 8 | Сборка тензора `X ∈ R^(12 × L_window)`, цели `y`, сохранение в `.npz` |

### Ключевые параметры

- `FS = 5000` — частота дискретизации (Гц)
- `FC = 25.0`, `ORDER = 4` — параметры эталонного ФВЧ Баттерворта (для справки)
- `T_PRE = 0.050`, `T_POST = 0.150` — окно анализа (сек)
- `LINE_PARAMS` — параметры линии для расчёта базовых величин (`Unom_kv`, `L_km`, `r1_ohm_km`, `x1_ohm_km`)
- `FAULT_DISTANCE_KM` — расстояние до КЗ в км (задаётся вручную в разделе 7)

## Сборка, запуск и тестирование

### Запуск проекта

1. Установить зависимости: `pip install -r requirements.txt`
2. Открыть `fault_location_preprocessing_pipeline.ipynb` в Jupyter Notebook / JupyterLab / VS Code.
3. Запустить все ячейки последовательно (Run All).
4. В разделе 7 при необходимости задать `FAULT_DISTANCE_KM` для расчёта целевой переменной `y`.

### Тестирование

Формального тестового фреймворка (pytest, unittest) в проекте нет. Валидация происходит через:

1. **Визуализацию** — после каждого шага строятся графики исходных/обработанных сигналов.
2. **Печать промежуточных значений** — выводятся ключевые метрики (амплитуды, отношения, ошибки детекции).
3. **Ручную проверку каналов** — функция `inspect_channels()` позволяет убедиться, что выбраны правильные номера каналов из CFG.

## Сбор и хранение научной литературы

При работе с `sources/` (загрузка статей для обзора литературы) придерживайся следующего workflow:

### 1. Поиск и первичный отбор
- Используй `tavily_search` или `web_search_exa` для поиска релевантных статей.
- Сохраняй список источников с DOI/URL в текстовом файле или markdown-таблице.

### 2. Эффективное скачивание (open-access)
- **MDPI, Preprints.org, arXiv, Frontiers:** используй `tavily_extract` с `extract_depth=advanced` и `maxCharacters=20000` (или больше). Запускай до 4–6 вызовов параллельно.
- **IEEE Xplore / institutional repositories:** иногда доступны прямые PDF — пробуй `fetch` с `max_length=80000`.
- **Сразу сохраняй результат** в `sources/XX_author_title.md` в виде структурированного markdown (Abstract → Methodology → Key Findings).

### 3. Paywalled / gated источники
- **ScienceDirect, Springer, Wiley, Elsevier:** автоматизированные скрипты на `requests` **не работают** (403 + CAPTCHA).
- Вместо бесплодных попыток скачать PDF, создавай `.md`-файл с:
  - Полным библиографическим описанием
  - Abstract и ключевыми словами
  - Кратким summary методологии и выводов (из preview / Google Scholar / tavily highlights)
  - DOI и примечанием о paywall

### 4. Недоступные или потерянные источники
- Если URL не резолвится (DNS failure, timeout), ищи альтернативные хосты через `web_search_exa`:
  - `site:researchgate.net`
  - `site:arxiv.org`
  - `site:semanticscholar.org`
  - `filetype:pdf "Title"`
- Если альтернативы нет — создавай `.md` с доступной информацией из поисковых highlights.

### 5. Логирование
- Веди `download_log.txt` с пометками `OK` / `PAYWALL` / `FAIL` для каждого источника. Это позволит быстро понять, что осталось дособирать вручную.
